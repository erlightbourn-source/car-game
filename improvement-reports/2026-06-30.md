## Research findings

1. **Near-miss reward feedback drives engagement and replay.** Burnout's "Near Miss" — passing
   extremely close to another vehicle at high speed without contact — is the same mechanic Lane
   Rush rewards. Near-miss/almost-win feedback engages reward pathways and extends sessions; this
   is exactly what Lane Rush's combo system models. Implication: the combo achievement deserves
   to be *visible* as a result, not just a transient in-run popup.
   Sources: [Burnout Wiki — Near Miss](https://burnout.fandom.com/wiki/Near_Miss),
   [Psychology of Games — The Near Miss Effect](https://www.psychologyofgames.com/2016/09/the-near-miss-effect-and-game-rewards/)
2. **Runners retain better when they give clear goals beyond raw score.** Endless runners have
   low baseline retention because difficulty ramps fast; the fix is clear objectives and
   status/competition signals (leaderboards, streaks) layered on top of scoring. Implication: a
   "best combo this run" stat on the game-over screen is a cheap, skill-based beat-your-streak
   goal that complements the existing leaderboard.
   Source: [Endless Runner Games — Core Mechanics and Top Titles](https://logicsimplified.com/newgames/endless-runner-games-core-mechanics-and-top-titles/)

## Shipped this run

- **`fdff12c` — Surface best near-miss combo on the game-over screen.** The engine already
  tracked `bestCombo` per run, but it only ever showed in the transient in-run HUD popup — the
  moment you crashed, your signature skill achievement vanished. Now the game-over screen reads
  `engine.bestCombo` and shows "🔥 Best combo: N× near-misses", only when a real streak (≥2) was
  built. Gives players a skill-based goal beyond raw score and reinforces the core near-miss hook.
  - UI-only, no balance change. Fairness before→after: normal 100%→100%, easy 100%→100% (PASS).
  - Cache version bumped 25 → 26.
  - Note: the working tree on entry already contained a partial version of this feature (CSS
    classes + HTML element + JS wiring staged unstaged). It was completed/verified and committed
    as one coherent change rather than discarded.

## Follow-on (shipped same day, human-authorized)

The combo-persistence item below was flagged for human review during the autonomous pass;
the owner then authorized it and it shipped:

- **`cc7d7b8` — Persist lifetime-best near-miss combo as a personal record.** The best combo a
  player has ever hit is now saved (`Shop.bestCombo`, in the existing localStorage save blob) and
  surfaced two ways: a third **COMBO** stat on the start screen next to BEST/COINS (visible on
  every return → durable beat-your-streak goal), and a **"— 🏆 record!"** flourish on the
  game-over combo line when a run sets a new lifetime best (≥2). A record only fires on a strict
  improvement of a real streak (≥2), so 0/1 combos never register and ties don't re-fire.
  - Verified: a `recordRun` unit test (record fires only on strict improvement; persists across a
    reload) + fairness gate. Fairness before→after: normal 100%→100%, easy 100%→100% (PASS).
  - Start-screen stat row reworked to `flex:1` + `flex-wrap` so three stats share the 340px panel
    without overflow (wraps safely on narrow phones). Cache version bumped 26 → 27.
  - Browser preview was unavailable this session (the project's preview port was held by another
    chat's dev server), so this was verified via the harness, the unit test, and a static layout
